# 兼容性策略

jsrc 采用语义化版本号。

## 正式版之前 (0.x)

不提供稳定性保证。CLI 参数、子命令名称、输出格式、默认值等均可能
随时变更。

## 1.0 之后

以下内容视为稳定，破坏性变更必须提升主版本号：

- CLI 结构：`jsrc <module> <subcommand> [options]`
- 文档中列出的长选项（如 --input、--output）
- 标准输出的表格文本、文档声明的 JSON 输出
- 以 Error: 开头输出至标准错误输出的错误信息
- 退出码：0（成功）、1（用法错误）、2（运行时错误）
- 文档中声明的输入文件格式（FASTA、GFF、CSV、TSV 等）

弃用流程：
1. 在 MINOR 版本中标记弃用，标准错误输出提示
2. 在下一个 MAJOR 版本中移除

以下内容始终可随时变更：帮助文本、未在 --help 中显示的选项、
不以 Error: 开头的诊断信息、性能表现、实验性模块、--verbose /
--debug 输出。

## 模块成熟度

每个模块在文档中标注成熟度等级：

- Stable — 适用所有 1.0 后的兼容性承诺
- Mature — 接口基本确定，待升级
- Experimental — 随时变更，不另行通知
- Deprecated — 计划移除
